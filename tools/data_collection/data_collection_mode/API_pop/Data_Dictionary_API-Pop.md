# Data Dictionary – API-Pop (MySQL Table: `rumor_reports`)

| Field Name             | Data Type     | Format / Size                          | Description     
|------------------------|---------------|----------------------------------------|-----------------------------------------------------------|
| `id_code`              | VARCHAR(36)   | UUID format                            | Autogenerated unique identifier for each form submission
| `report_opened`        | DATETIME      | `YYYY-MM-DD HH:MM:SS`                  | Timestamp when user clicked "Report New Case"  
| `consultation_date`    | DATE          | `YYYY-MM-DD`                           | Date of consultation  
| `sex`                  | CHAR(1)       | `M` / `F` / `Other` / `Rather not say` | Biological sex of the individual   
| `birth_date`           | DATE          | `YYYY-MM-DD`                           | Date of birth                            
| `residence_country`    | VARCHAR(100)  | -                                      | Country of residence
| `residence_province`   | VARCHAR(100)  | -                                      | Province/state of residence 
| `residence_department` | VARCHAR(100)  | -                                      | Department of residence
| `residence_locality`   | VARCHAR(100)  | -                                      | Locality of residence
| `censal_fraction`      | VARCHAR(100)  | -                                      | Censal Fraction of residence
| `symptom_onset_date`   | DATE          | `YYYY-MM-DD`                           | Date when symptoms began
| `symptom_description`  | TEXT          | -                                      | Free text description of symptoms     
| `suspected_diagnosis`  | TEXT          | -                                      | Free text of suspected diagnosis             
| `confirmation_time`    | DATETIME      | `YYYY-MM-DD HH:MM:SS`                  | Timestamp when user submitted the form
| `email     `           | VARCHAR(36)   |                                        | Professionals email     


---

**Note:** All fields are stored in the `rumor_reports` table in MySQL. `id_code` acts as the primary key and must be unique.
